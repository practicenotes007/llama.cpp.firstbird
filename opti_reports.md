## 2. 推理理论上限（基于实测带宽）

### 公式

```
tok/s ≈ DDR_BW (GB/s) / 模型大小 (GB) × 0.85
```

- `DDR_BW`: 实测 Triad 多线程带宽 ≈ 21.46 GB/s
- `× 0.85`: 经验系数，扣除 KV cache 读写、非矩阵乘算子（norm/silu/softmax/rope）和内存分配等开销

## 3. 瓶颈根因

### 3.1 权重体积远超缓存

| 缓存层级 | 容量 | 单层权重 |
|----------|------|----------|
| L1 D$ / 核 | 64 KB | ~108 MB |
| L2 / 核 (A76) | 512 KB | ~108 MB |
| L3 共享 | 2-4 MB | ~108 MB |
| DDR4 | 15.5 GB | 4.0 GB (全部) |

LLaMA 7B Q4_0 4.0 GB 权重远超任何一级缓存。decode 阶段每次生成一个 token，所有权重必须从 LPDDR4X 读取一次，无一命中缓存。

### 3.2 代码级原因（ggml 早期版本）

1. **无层间流水线**: `ggml_graph_compute()` 节点之间串行，计算层 N 时层 N+1 的权重仍在 DDR 中
2. **无预取指令**: `ggml_vec_dot_q4_0` NEON 内核没有 `__builtin_prefetch`
3. **未启用 SDOT**: ARM v8.2 dotprod 扩展未利用

## 4. 优化方案汇总（修正后预期）

| 方案 | 预期加速 | 改动量 | 优先级 | 4.0 GB 模型预期 |
|------|----------|--------|--------|------------------|
| A: NEON 内核预取 | 1.1-1.3x | 小 | P0 | 4.6 → 5.1-6.0 |
| B: ARM SDOT 指令 | 1.5-2.0x | 中 | P0 | 4.6 → 6.9-9.2 |
| C: 层间流水线预取 | 1.2-1.4x | 中 | P1 | 与 A/B 叠加 |
| D: 线程调度交错化 | 1.1-1.2x | 小 | P1 | 与 A/B 叠加 |

> 注意：所有方案叠加后也**不会超过 4.6 tok/s × 2.0 = ~9.2 tok/s**。如果不满足，必须考虑方案 F（换更小模型或更激进量化）。

### 方案 F（减少搬运量）

| 子方案 | 预期收益 | 风险 |
|--------|----------|------|
| F2: Q2_K 量化 | 4.0 GB → 2.3 GB，理论上限 ~8.0 tok/s | 精度损失 |
| F2: IQ3_XS 量化 | 4.0 GB → 3.0 GB，理论上限 ~6.1 tok/s | 精度损失较小 |
| 换小模型 | Qwen2.5-1.5B ≈ 0.9 GB，理论上限 ~20 tok/s | 能力下降 |
