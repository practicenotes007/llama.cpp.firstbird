### 模型格式判断

根据文件名 `llama7b.hf.tar.gz` 可知这是 **HuggingFace 格式**（不是 Meta 原始格式），需要走"路径 B"转换流程：

1. HuggingFace 格式 → Meta 原始格式（使用 `convert_hf_to_pth.py` 或 `convert_hf_to_pth_lowmem.py`）

### 转换服务器环境准备（补充）

> ⚠️ **中国大陆服务器特殊说明**：以下操作涉及访问外网资源，在中国大陆服务器上需要配置国内镜像源，否则会安装失败。

#### 1. 配置 apt 国内镜像（Ubuntu 系统包）

```bash

sudo sed -i 's@//.*security.ubuntu.com@//mirrors.aliyun.com@g' /etc/apt/sources.list

# 更新软件包列表
sudo apt-get update
```

#### 2. 安装编译工具链

```bash
sudo apt-get install -y build-essential
```

#### 3. 配置 pip 国内镜像（Python 包）

```bash

cat > ~/.pip/pip.conf << 'EOF'
```

## 模型格式转换（路径 B：HF → GGML → 量化）

> 以下所有命令均在 **"模型转换-实例信息"** 服务器（172.31.123.130）上执行。

### 步骤 0：确认解压后的目录结构

```bash
cd ~/llama.cpp.firstbird
ls -la
# 确认存在 convert_hf_to_pth.py、convert_hf_to_pth_lowmem.py、convert-pth-to-ggml.py、quantize.cpp 等
```

解压后的 HF 模型文件可能在子目录中（例如 `llama7b/`），执行以下命令确认：

```bash
# 查找 config.json 所在目录（即 HF 模型根目录）
find . -name "config.json" -not -path "./.git/*" 2>/dev/null
# 假设输出为 ./llama7b/config.json，则 HF_MODEL_DIR=./llama7b
```

根据上一步的结果设置变量（本文以 `./llama7b` 为例）：

```bash
HF_MODEL_DIR=./llama7b
ls "$HF_MODEL_DIR"
# 应看到: config.json  pytorch_model-*.bin  pytorch_model.bin.index.json  tokenizer.model 等
```

---

### 步骤 1：HF 格式 → Meta 原始格式

二选一执行：

#### 方式 A：标准转换（峰值内存 ~20 GB，内存 ≥ 32 GB 推荐）

```bash
python3 convert_hf_to_pth.py "$HF_MODEL_DIR" ./models/7B
```

**说明**：用 `transformers.AutoModelForCausalLM.from_pretrained()` 一次性加载全部权重到内存，速度最快。

#### 方式 B：低内存转换（峰值内存 ~3 GB，内存不足时使用）

```bash
python3 convert_hf_to_pth_lowmem.py "$HF_MODEL_DIR" ./models/7B
```

**说明**：逐张量读取 `.safetensors` 文件，内存友好。需要 HF 目录中存在 `model.safetensors.index.json` 和对应的 `.safetensors` 分片文件。

#### 验证输出

```bash
ls -lh ./models/7B/
# 预期输出：
# consolidated.00.pth   (~12.5 GB)
# params.json           (~200 B)
```

---

### 步骤 2：复制 tokenizer.model 到正确位置

`convert-pth-to-ggml.py` 硬编码读取 `models/7B/../tokenizer.model`，需要将 tokenizer 放在 `models/` 目录下：

```bash
cp "$HF_MODEL_DIR/tokenizer.model" ./models/tokenizer.model
ls -lh ./models/tokenizer.model
# 预期约 500 KB
```

---

### 步骤 3：Meta 原始格式 → GGML FP16

```bash
cd ~/llama.cpp.firstbird
python3 convert-pth-to-ggml.py ./models/7B 1
```

**参数说明**：

| 参数 | 值 | 含义 |
|------|-----|------|
| `dir_model` | `./models/7B` | 包含 `consolidated.00.pth` + `params.json` 的目录 |
| `ftype` | `1` | `0` = float32（~26 GB），`1` = float16（~13 GB） |

**输出文件**：`./models/7B/ggml-model-f16.bin`（约 13 GB）

**验证**：

```bash
ls -lh ./models/7B/ggml-model-f16.bin
# 预期约 13 GB

# 验证魔数（前 4 字节应为 "ggml" 即 0x67676d6c）
xxd -l 4 ./models/7B/ggml-model-f16.bin
# 预期输出: 00000000: 6767 6d6c                            ggml
```

---

### 步骤 4：量化（Q4_0 / Q4_1）

```bash
cd ~/llama.cpp.firstbird

# Q4_0 量化（推荐，文件更小）
./quantize ./models/7B/ggml-model-f16.bin ./models/7B/ggml-model-q4_0.bin 2

# 或 Q4_1 量化（精度稍高，文件稍大）
# ./quantize ./models/7B/ggml-model-f16.bin ./models/7B/ggml-model-q4_1.bin 3
```

**量化类型对比**：

| 类型 | itype | 输出大小 | 精度 | 适用场景 |
|------|-------|---------|------|----------|
| Q4_0 | 2 | ~3.8 GB | 较低 | 资源受限设备，追求最小体积 |
| Q4_1 | 3 | ~4.2 GB | 较高 | 追求更高推理质量 |

**验证**：

```bash
ls -lh ./models/7B/ggml-model-q4_0.bin
# 预期约 3.8 GB
```

---

### 步骤 5：运行推理（验证转换成功）

```bash
cd ~/llama.cpp.firstbird

./main \
    -m ./models/7B/ggml-model-q4_0.bin \
    -t 8 \
    -n 128 \
    --temp 0.8 \
    --top_k 40 \
    --top_p 0.95 \
    -p "Building a website can be done in 10 simple steps:"
```

**参数说明**：

| 参数 | 值 | 含义 |
|------|-----|------|
| `-m` | 模型路径 | Q4_0 量化后的权重文件 |
| `-t` | `8` | 推理线程数（16 核服务器建议 8-16） |
| `-n` | `128` | 最大生成 token 数 |
| `--temp` | `0.8` | 采样温度（越高越随机） |
| `--top_k` | `40` | Top-K 采样 |
| `--top_p` | `0.95` | Nucleus 采样 |
| `-p` | prompt | 输入提示词 |

如果推理正常输出文本，说明整个转换流程成功。

### 一键转换脚本（从解压后直接跑到量化完成）

如果环境已准备好（编译完成、依赖已装），可以将以上步骤合并为一个脚本：

```bash
#!/bin/bash
set -e

cd ~/llama.cpp.firstbird

# 0. 找到 HF 模型目录
HF_MODEL_DIR=$(find . -maxdepth 2 -name "config.json" -not -path "./.git/*" 2>/dev/null | head -1 | xargs dirname)
if [ -z "$HF_MODEL_DIR" ]; then
    echo "ERROR: 找不到 config.json，请确认模型已解压"
    exit 1
fi
echo "HF_MODEL_DIR: $HF_MODEL_DIR"

# 判断内存是否充裕（≥32 GB 用标准版，否则用低内存版）
MEM_GB=$(free -g | awk '/^Mem:/{print $2}')
if [ "$MEM_GB" -ge 30 ]; then
    echo "内存充足 ($MEM_GB GB)，使用标准转换..."
    python3 convert_hf_to_pth.py "$HF_MODEL_DIR" ./models/7B
else
    echo "内存紧张 ($MEM_GB GB)，使用低内存转换..."
    python3 convert_hf_to_pth_lowmem.py "$HF_MODEL_DIR" ./models/7B
fi

# 1. 复制 tokenizer
cp "$HF_MODEL_DIR/tokenizer.model" ./models/tokenizer.model

# 2. FP16 转换
python3 convert-pth-to-ggml.py ./models/7B 1

# 3. Q4_0 量化
./quantize ./models/7B/ggml-model-f16.bin ./models/7B/ggml-model-q4_0.bin 2

# 4. 验证
echo "=== 输出文件 ==="
ls -lh ./models/7B/ggml-model-*.bin

echo ""
echo "转换完成！推理命令："
echo "./main -m ./models/7B/ggml-model-q4_0.bin -t 8 -n 128 -p \"Hello, world!\""
```

将以上脚本保存为 `~/llama.cpp.firstbird/convert_all.sh`，然后执行：

```bash
chmod +x ~/llama.cpp.firstbird/convert_all.sh
~/llama.cpp.firstbird/convert_all.sh
```
